# Firmware

Firmware is intended to live in a separate repository so the hardware project stays focused on schematic capture, PCB layout, simulations, and documentation.

A future firmware repository would likely contain:

- STM32 motor control and sensor acquisition code
- LoRa telemetry framing
- Raspberry Pi CM4 interface services
- Bring-up and production test utilities

This folder remains as a placeholder to show the intended split between hardware and software deliverables.
